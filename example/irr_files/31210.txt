remarks:        -------------------------- BGP Routing Policy ------------------------------
remarks:        DTEL-IX Route Servers support following RFC and Drafts:
remarks:        * RFC 1997              BGP Communities Attribute;
remarks:        * RFC 4360              BGP Extended Communities Attribute;
remarks:        * RFC 4384              BGP Communities for Data Collection;
remarks:        * RFC 4893, 6793        BGP Support for Four-Octet Autonomous System (AS) Number Space;
remarks:        * RFC 7947              Internet Exchange BGP Route Server;
remarks:        * draft-hilliard-ix-bgp-route-server-operations-03;
remarks:
remarks:        Each of route-server DOES NOT modify, process or remove
remarks:        communities received from customers except those communities
remarks:        which are processed by the route-servers themselves:
remarks:        0:*, 31210:*, 65000..65007:*
remarks:
remarks:        = Communities accepted from customers =
remarks:
remarks:        * Communities affecting announces to all peers:
remarks:        0:31210         - Do not announce route to all peers
remarks:        31210:31210     - Announce routes to all peers
remarks:
remarks:        * Communities affecting announces to certain peer:
remarks:        0:X             - Do not announce route to peer ASX
remarks:        31210:X         - Announce route to peer ASX only
remarks:
remarks:        If peer's AS is 32-bit then BGP extended community attribute
remarks:        should be used:
remarks:        rt:0:X          - Do not announce route to peer ASX
remarks:        rt:31210:X      - Announce route to peer ASX only
remarks:
remarks:        RFC1997 communities 'no-export' and 'no-advertise' are also
remarks:        supported. Other RFC1997 well-known communities are not
remarks:        supported and stripped.
remarks:
remarks:        Global blocking community 0:31210 has the highest precedence.
remarks:        Thus if a route have both 0:31210 and 31210:X then it will
remarks:        not be announced to any peer.
remarks:
remarks:        * Blackholing
remarks:        Only /32 accepted for blackholing. In order to blackhole
remarks:        certain /32 prefix customer should add communities 0:0 and 31210:X
remarks:        where X is an AS number of peer from which traffic should be
remarks:        blackholed. If X is 31210 then traffic will be blackholed from
remarks:        all peers.
remarks:
remarks:        = Communities sent to customers =
remarks:
remarks:        * Information about source ASN:
remarks:        31210:X, ro:31210:X - Route was received from ASX
remarks:
remarks:        * Information about contry (gained from Maxmind GeoIP Database):
remarks:        6500M:10CCC - Network is used in country CCC on continent M, where:
remarks:        M is 1-digit continet code as it is defined in RFC4384:
remarks:        1       -> Africa (AF)
remarks:        2       -> Oceania (OC)
remarks:        3       -> Asia (AS)
remarks:        4       -> Antarctica (AQ)
remarks:        5       -> Europe (EU)
remarks:        6       -> Latin America/Caribbean Islands (LAC)
remarks:        7       -> North America (NA)
remarks:        CCC is 3-digit country code as it is defined in ISO3166;
remarks:        ... if information is available in GeoIP Database.
remarks:        Examples:
remarks:        65005:10804 - Ukraine
remarks:        65005:10616 - Poland
remarks:        65006:10132 - Argentina
remarks:        65007:10124 - Canada
remarks:        If there is no information about exact country in GeoIP database
remarks:        then following communities are applied:
remarks:        65000:10000 - Neither country nor region is known;
remarks:        65003:10000 - Asian/Pacific Region;
remarks:        65005:10000 - European Union;
remarks:        65008:10000 - Anonymous Proxy;
remarks:        65009:10000 - Satellite Provider/Anycast.
remarks:        See http://dev.maxmind.com/geoip/legacy/codes/iso3166/ for more details.
